# Opus 工作流实战指南

> **难度**：中级 | **阅读时间**：15 分钟

## 前言

很多人使用 AI 时容易停留在“单次提问”。本教程的目标是把一次任务拆成可复用流程，让模型输出更稳定、可验收、可复盘。

## 一、任务拆解模板

开始前先定义 3 个要素：

- **目标**：这次要交付什么结果？
- **边界**：不能做什么？有哪些约束？
- **验收标准**：什么样算完成？

可直接复用：

```text
你现在是我的项目助理。
目标：{填写目标}
边界：{填写约束与禁区}
输出格式：{Markdown / 表格 / JSON}
验收标准：{可量化条目}
请先给出执行计划，再逐步产出结果。
```

## 二、提示词结构化

建议把提示词分成四段：

1. 角色与目标
2. 输入背景（数据、上下文）
3. 输出格式与长度
4. 质量检查清单

示例：

```text
角色：资深产品分析师
任务：把下面的需求整理成 PRD 草稿
输入：{贴入需求原文}
输出：
- 用户故事
- 功能清单
- 风险与假设
- 里程碑（按周）
质量检查：
- 术语统一
- 无重复项
- 每条功能都有验收口径
```

## 三、结果复盘与二次迭代

首版结果完成后，不要直接发布，建议再跑一次复盘：

- 有无遗漏关键约束
- 是否存在模糊表述
- 是否可直接执行

复盘提示词：

```text
请你作为审稿人，对上面的结果做质量审查：
1) 找出3个最可能导致执行失败的问题
2) 给出逐条修正建议
3) 输出修正版
```

## 四、常见问题

### Q1：模型输出不稳定怎么办？

优先检查输入是否完整、输出格式是否明确、验收标准是否可量化。

### Q2：如何让团队成员复用同一套方法？

把模板沉淀到团队知识库，统一字段与检查清单。

### Q3：是否要一次把所有信息都给模型？

不建议。先给核心上下文，按阶段补充，可显著提升可控性。

## 总结

真正提升效率的不是“更复杂的提问”，而是可复用的流程。你可以先把这套模板应用到一个小任务，再扩展到完整项目。
